System basics

Getting Started

Welcome to the Wormhole Queue System wiki. This guide explains how the site works for students, Wormhole Assistants, and administrators. The system is designed to help students request assistance, allow staff to manage requests clearly, and keep the queue visible in real time.

Important: students do not need to log in to request help. Accounts are mainly for staff and administrators.

Creating your account

  1. Open the User menu in the top navigation bar.
  2. Select Log In.
  3. Enter your assigned username or email and password.
  4. After login, staff-only tools will appear in the User menu and profile pages.

What each role can do

Students

  • Submit online help requests
  • Request in-person help from table hardware
  • Watch the live queue
  • Open schedule and meeting resources from Home

Wormhole Assistants

  • Log in and claim tickets
  • Work active requests
  • Resolve completed tickets
  • Monitor queue activity

Administrators

  • Create and manage staff users
  • Use queue maintenance tools
  • Export archive data
  • Support troubleshooting and oversight

Using the queue

Student Guide

Getting help online

  1. Go to the Home page.
  2. In the For Students section, open the help request form.
  3. Enter a name the Wormhole Assistant will recognize on your platform of choice.
  4. Select your class from the dropdown menu.
  5. Choose the platform the assistant should use to reach you.
  6. Submit the request.
  7. You will be redirected to the Live Queue page so you can monitor your position.

If you need the online meeting links, return to the Home page and use the student resource section to access the active Zoom or Teams links for the Wormhole.

Getting help in person

  1. Go to an available table with a button box.
  2. Press the button for the class you need help with.
  3. A ticket is added to the queue with your table and course.
  4. The ticket appears on the live queue display.
  5. A Wormhole Assistant will come to your table when they are available.

Understanding queue status

Numbered position

Your ticket is waiting in line and has not yet been claimed by an assistant.

IN PROGRESS

An assistant has already claimed your ticket and is currently working with you.

Removed from queue

The request has been resolved or cleared from the active queue.

Student tips

  • Use a recognizable display name for online help.
  • Keep the Live Queue page open after submitting a request.
  • Stay available on the contact platform you selected.
  • If you no longer need help, notify staff if possible.

The User menu provides access to login, profile pages, hardware tools, registration, queue dashboard tools, and logout actions depending on role permissions.

What users should expect

  • Students can use the request and queue pages without needing staff credentials.
  • Staff tools appear only after successful login.
  • Administrators see additional management and export tools.
  • The site should prioritize quick navigation and current queue visibility.

Staff workflow

Wormhole Assistants

Assistant workflow

  1. Log in through the User menu.
  2. Open your profile or queue dashboard page.
  3. Claim the next available ticket using the queue controls.
  4. Contact the student or go to the student’s table.
  5. Work the request.
  6. Resolve the ticket once the interaction is complete.

Claiming tickets

When an assistant claims the next ticket, the ticket should leave the waiting state and appear as IN PROGRESS on the live queue. This keeps the queue accurate for both students and other staff members.

Best practices

  • Claim tickets promptly when available.
  • Communicate clearly with students, especially for online requests.
  • Resolve tickets only after the interaction is actually complete.
  • Use skip, clear, or maintenance tools carefully.
  • Keep queue information accurate so students can trust the display.

Management tools

Admin Features

User management

Administrators can register staff accounts, manage permissions, and maintain access to the system. This ensures that only authorized personnel can use queue management tools.

Queue tools

  • Monitor the current queue state
  • Clear or reset queue information when needed
  • Support staff during live operations
  • Validate that tickets are transitioning correctly between waiting, in progress, and resolved states

Archive and export actions

Administrative views may include archive export or review tools that allow historical ticket information to be saved, examined, or reported. These tools are useful for documentation, oversight, and system review.

Administrative caution: queue reset, flush, or export actions should be used carefully during active support sessions because they can affect live student requests.

Common issues

Troubleshooting

I submitted a request, but I do not see it in the queue

  • Refresh the Live Queue page.
  • Confirm the request form was submitted successfully.
  • Check that all required fields were filled in.
  • Confirm that the ticket is not already marked IN PROGRESS.

The Live Queue is not updating

  • Refresh the page or hard refresh your browser.
  • Check whether JavaScript files are loading correctly.
  • Inspect the browser console for MIME type or script errors.
  • Ask an administrator to verify server-side queue events.

The User menu does not open

  • Refresh the page.
  • Verify that JavaScript is enabled.
  • Check for console errors related to nav-menu.js.

I cannot log in

  • Check your username or email.
  • Verify the password.
  • Confirm that your account was created correctly.
  • Contact an administrator if the issue continues.

Wiki navigation links are not working

  • Refresh the page.
  • Confirm that the section anchors match the link targets.
  • Check the console for JavaScript selector errors.

Styles or scripts are not loading

  • Verify that static files are being served correctly.
  • Check the browser console for content-type or MIME errors.
  • Confirm that CSS and JS files are loading from the correct project path.

Frequently asked questions

FAQ

Do students need to log in to request help?

No. Students can submit requests without logging in.

How do I know where I am in line?

Use the Live Queue page after submitting a request. Waiting tickets show a queue position, while active claims show IN PROGRESS.

What does “IN PROGRESS” mean?

It means an assistant has already claimed the ticket and is actively working with that student.

Can I request help online and in person?

Yes. The system supports online requests and in-person help through table hardware.

What happens when my ticket is resolved?

The ticket leaves the active queue. Depending on system configuration, the request may still be stored in archived records.

Who can manage users?

Administrators are responsible for user creation, permission changes, and staff account maintenance.
